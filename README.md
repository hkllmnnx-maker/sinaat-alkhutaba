# صِناعةُ الخُطَباء

## نظرة عامة على المشروع
- **الاسم**: صناعة الخطباء (Sina'at al-Khutaba)
- **الهدف**: موقع إسلامي تعليمي تفاعلي لتأهيل الخطباء، يقدّم دورة مكثفة ومنظمة في فنّ الخطابة والإلقاء بمحتوى علمي رصين مناسب لجميع المستويات.
- **الشعار/الهوية**: مستوحى من شعار المجموعة (أزرق داكن `#0F569B` + بني ذهبي)، يجمع بين الأصالة والمعاصرة.

## الميزات المكتملة ✅
- **صفحة رئيسية احترافية** بواجهة عصرية (Hero, إحصاءات, الأهداف الأربعة, المميزات, عرض الوحدات, نداء للعمل).
- **دورة تعليمية متكاملة** من **4 وحدات** و**12 درسًا**:
  1. مدخل إلى فن الخطابة (التعريف، الأركان، صفات الخطيب).
  2. بناء الخطبة وإعدادها (اختيار الموضوع، الهيكلة، الصياغة).
  3. مهارات الإلقاء والأداء (الصوت، لغة الجسد، التغلب على الخوف).
  4. الإقناع والتأثير وصناعة القائد (أساليب الإقناع، المواقف الطارئة، صناعة القائد).
- **أنشطة وتمارين** (تطبيقية/تأملية/كتابية/صوتية) في كل درس.
- **اختبارات تفاعلية** بعد كل درس مع تصحيح فوري وشرح.
- **خرائط ذهنية ومشجرات** (4 خرائط بصرية) لأركان الخطابة والبناء والأداء والإقناع.
- **معرض مشجّرات مصوّر غني لكل درس** (**36 خريطة ذهنية مصوّرة** — **3 مشجّرات لكل درس**) مولّدة بنموذج OpenAI Images 2.0 (gpt-image-2): مشجّر «نظرة شاملة» + مشجّران يفصّلان موضوعَين فرعيَّين في الدرس. نصوص عربية دقيقة **مُشكّلة بالكامل** ومتحقَّق منها لغويًّا ونحويًّا وعلميًّا (تشكيل/إملاء/مضمون)، بتصميم إنفوجرافيك موحّد جذّاب (خلفية عاجية بزخرفة إسلامية، عقدة عنوان كحلية، بطاقات ذهبية، عقد فرعية زرقاء فاتحة). تُعرض داخل كل صفحة درس في **معرض شبكي متجاوب** مع عنوان ووصف لكل مشجّر وإمكانية التكبير بالنقر. الملفات في `public/static/diagrams/{lessonId}.png` و`{lessonId}-b.png` و`{lessonId}-c.png` (مضغوطة ~200KB لكل صورة لأداء سريع).
- **نظام التدرّج في الدراسة (Sequential Gating)**: تُدرَس الدورة بالترتيب درسًا درسًا — الدرس الأول مفتوح فقط، وكل درس تالٍ يبقى **مقفلًا** حتى يُكمل المتعلّم الدرس الذي قبله. في كل درس **يختار المتعلّم مدّة الدراسة** من **خيارات متعددة (من دقيقة واحدة إلى خمس دقائق)** ثمّ يضغط **«بدء الدرس»** فيبدأ **مؤقّت تنازلي بالمدّة المختارة**. لا يُفتح زرّ الانتقال للدرس التالي إلّا بتحقّق **شرطين معًا**: (١) مرور المدّة المختارة منذ بدء الدرس، (٢) **الإجابة الصحيحة عن جميع أسئلة الاختبار** (يُسمح بإعادة المحاولة عند الخطأ). تُحفظ المدّة المختارة لكل درس في `localStorage` (مفتاح `sk_duration_v1`)، فلا تتغيّر عند إعادة تحميل الصفحة. كما يُمنَع الوصول المباشر عبر الرابط لأي درس مقفل (إعادة توجيه تلقائية للدرس المتاح)، وتظهر أيقونة قفل على الدروس غير المتاحة في القوائم.
- **وضع المشرف (زرّ مخفي على الشعار)**: عند **النقر المزدوج** على شعار الموقع أعلى اليمين (أو **الضغط المطوّل** على الجوال) تظهر نافذة تسجيل دخول؛ بإدخال اسم المستخدم `Adman5` وكلمة السر `12734` تُفتح **جميع الدروس** ويُرفع كل قيد أو قفل (تجاوز شرطَي المدّة والاختبار، وفتح التنقّل الحرّ). تظهر شارة خضراء «وضع المشرف مُفعّل» مع زرّ خروج. **جلسة المشرف مؤقّتة**: تنتهي تلقائيًا عند إغلاق الموقع/التبويب (مخزّنة في `sessionStorage`) أو بعد **خمول طويل (٣٠ دقيقة)**، وعندها تعود القيود الطبيعية. أمّا **تقدّم الدارس (الدروس المكتملة) فيُحفظ دائمًا ولا يُفقد** في `localStorage`.
- **تتبّع التقدّم** تلقائيًا عبر `localStorage` (شريط تقدّم + علامات إتمام الدروس).
- **شريط تقدّم القراءة** + تأثيرات ظهور عند التمرير + تصميم متجاوب بالكامل (RTL).

## خريطة المسارات (URIs)
| المسار | الوصف |
| --- | --- |
| `/` | الصفحة الرئيسية |
| `/course` | صفحة الدورة (كل الوحدات والدروس مع شريط التقدّم) |
| `/module/:id` | صفحة وحدة واحدة (مثال: `/module/m1`) |
| `/lesson/:id` | صفحة درس واحد (مثال: `/lesson/m1l1`) — محتوى + أنشطة + اختبار |
| `/mindmaps` | الخرائط الذهنية والمشجرات |
| `/about` | عن المجموعة ومحاورها الأربعة |
| `/static/*` | الأصول الثابتة (الشعار، CSS، JS) |

معرّفات الدروس: `m1l1..m1l3`, `m2l1..m2l3`, `m3l1..m3l3`, `m4l1..m4l3`.

## بنية البيانات
- **المحتوى التعليمي**: معرّف بالكامل في `src/data/course.ts` (Modules → Lessons → Sections/Exercises/Quiz + Diagrams + MindMaps + Goals). كل درس يحوي حقل `diagrams: Diagram[]` (src/title/caption) لثلاث مشجّرات مصوّرة تُعرض عبر معرض في `src/pages/lesson.tsx`.
- **التخزين**:
  - `localStorage` — مفتاح `sk_progress_v1` (الدروس المكتملة، يُحفظ دائمًا)، ومفتاح `sk_started_v1` (أوقات بدء كل درس)، ومفتاح `sk_duration_v1` (المدّة المختارة لكل درس بالدقائق ١–٥ لحساب المؤقّت التنازلي).
  - `sessionStorage` — مفتاح `sk_admin_session_v1` (جلسة المشرف المؤقّتة مع طابع آخر نشاط؛ تنتهي عند الخروج أو بعد ٣٠ دقيقة خمول).
  - لا حاجة لقاعدة بيانات خادمية. ترتيب الدروس يُمرَّر لواجهة المتصفّح عبر السمة `data-lesson-order` على `<body>`.
- **تدفّق البيانات**: بيانات ثابتة (static) تُعرض عبر Hono SSR (JSX) → HTML، والتفاعل من جانب العميل عبر `public/static/app.js`.

## دليل الاستخدام
1. افتح الصفحة الرئيسية وتعرّف على الأهداف والمميزات.
2. اضغط **«ابدأ الدورة»** للانتقال إلى صفحة الدورة ورؤية كل الوحدات.
3. ادخل أي درس: اقرأ المحتوى، نفّذ الأنشطة، ثم أجب على الاختبار القصير.
4. اضغط **«أنهيت هذا الدرس»** (أو أكمل القراءة) لتُحتسب ضمن تقدّمك.
5. راجع **الخرائط الذهنية** لتثبيت المفاهيم بصريًّا.

## التقنيات
- **Hono** (إطار خفيف على Cloudflare) + **JSX SSR**.
- **Vite** للبناء، **Wrangler** للتطوير/النشر.
- **Tailwind غير مستخدم** — نظام CSS مخصّص (`public/static/style.css`) + Font Awesome + خطوط Google (Tajawal/Cairo).

## حلّ مشكلة الحجب في بعض الدول (اليمن وغيرها)
نطاقات `*.pages.dev` و`github.io` قد تكون محجوبة على مستوى مزوّدي الإنترنت في بعض الدول مثل اليمن — وهذا حجب خارجي لا يمكن التحكّم به من داخل الكود (يُنفَّذ غالبًا بفحص اسم النطاق في طبقة SNI/DNS). الحلّ يعتمد على إتاحة **روابط بديلة على نطاقات مختلفة**:

### ١) بروكسي Cloudflare Worker (الموصى به) ⭐
👉 **https://khutaba-proxy.khutaba-portal.workers.dev**

- نطاق `workers.dev` مختلف عن `pages.dev`، وقد لا يكون مدرجًا في قوائم الحجب.
- يعمل كبروكسي عكسي: المتصفّح يتصل بنطاق الـ Worker، والـ Worker (داخل شبكة Cloudflare) يجلب الصفحة من الموقع الأصلي ويعيدها — الاتصال بالأصل من خادم لخادم لا يمرّ عبر مزوّد الإنترنت لدى الزائر فلا يُحجب.
- يجلب المحتوى الحيّ تلقائيًا، فلا يحتاج إعادة نشر عند تحديث الموقع.
- الكود في مجلّد `proxy-worker/`. للتحديث: `cd proxy-worker && npx wrangler deploy`.

### ٢) نسخة GitHub Pages الثابتة
👉 **https://hkllmnnx-maker.github.io/sinaat-alkhutaba/**

- نسخة ثابتة تُولَّد عبر `npm run build:static` إلى مجلّد `docs/`، تُنشَر من فرع `main` مسار `/docs`.
- يدعم `app.js` المسار الفرعي تلقائيًا (`BASE_PATH`).

### ٣) الحلّ الدائم النهائي: دومين مخصص
لضمان عدم الحجب نهائيًا، يُنصح بشراء **دومين مخصص فريد** (مثل `.com`) وربطه بمشروع Cloudflare Pages — النطاقات الفريدة غير المعروفة لا تكون في قوائم الحجب. (راجع قسم «ربط دومين مخصص» في الأسفل عند توفّر دومين.)

## ربط دومين مخصص (عند توفّره)
عند شراء دومين (مثلًا `example.com`) أضِفه إلى Cloudflare ثم اربطه بالمشروع:
```bash
# عبر لوحة Cloudflare: Pages → sinaat-alkhutaba → Custom domains → Set up a domain
# أو عبر API (بعد إضافة الدومين كـ zone في الحساب):
export CLOUDFLARE_API_TOKEN="<token>"
npx wrangler pages deployment list --project-name sinaat-alkhutaba
# ثم أضف الدومين من لوحة التحكم وسيُصدر Cloudflare شهادة SSL تلقائيًا.
```

## آخر الإصلاحات
- **إصلاح اختفاء نافذة المشرف**: كانت النقرة المفردة على شعار الموقع (رابط `<a href="/">`) تُحدِث انتقالًا/إعادة تحميل للصفحة فتختفي النافذة عند اكتمال التحميل. الحلّ: منع السلوك الافتراضي للرابط دائمًا (`preventDefault`) واستخدام مؤقّت 280ملّي ثانية للتمييز بين النقرة المفردة (انتقال) والمزدوجة (فتح النافذة دون انتقال) — فأصبحت النافذة **تبقى ولا تختفي**.
- **إصلاح الحجب**: إضافة نسخة GitHub Pages البديلة (انظر القسم أعلاه).
