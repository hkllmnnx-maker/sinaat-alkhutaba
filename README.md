# صِناعةُ الخُطَباء

## نظرة عامة على المشروع
- **الاسم**: صناعة الخطباء (Sina'at al-Khutaba)
- **الهدف**: موقع إسلامي تعليمي تفاعلي لتأهيل الخطباء، يقدّم دورة مكثفة ومنظمة في فنّ الخطابة والإلقاء بمحتوى علمي رصين مناسب لجميع المستويات.
- **الشعار/الهوية**: مستوحى من شعار المجموعة (أزرق داكن `#0F569B` + بني ذهبي)، يجمع بين الأصالة والمعاصرة.

## الميزات المكتملة ✅
- **صفحة رئيسية احترافية** بواجهة عصرية (Hero, إحصاءات, الأهداف الأربعة, المميزات, عرض الوحدات, نداء للعمل).
- **دورة تعليمية متكاملة** من **4 وحدات** و**12 درسًا**:
  1. مدخل إلى فن الخطابة (التعريف، الأركان، صفات الخطيب).
  2. بناء الخطبة وإعدادها (اختيار الموضوع، الهيكلة، الصياغة).
  3. مهارات الإلقاء والأداء (الصوت، لغة الجسد، التغلب على الخوف).
  4. الإقناع والتأثير وصناعة القائد (أساليب الإقناع، المواقف الطارئة، صناعة القائد).
- **أنشطة وتمارين** (تطبيقية/تأملية/كتابية/صوتية) في كل درس.
- **اختبارات تفاعلية** بعد كل درس مع تصحيح فوري وشرح.
- **خرائط ذهنية ومشجرات** (4 خرائط بصرية) لأركان الخطابة والبناء والأداء والإقناع.
- **تشجير مصوّر لكل درس** (12 خريطة ذهنية مصوّرة) مولّدة بنموذج OpenAI Images 2.0 (gpt-image-2) — نصوص عربية دقيقة مُشكّلة، تصميم إنفوجرافيك جذّاب، تُعرض داخل كل صفحة درس مع إمكانية التكبير. الملفات في `public/static/diagrams/{lessonId}.png`.
- **تتبّع التقدّم** تلقائيًا عبر `localStorage` (شريط تقدّم + علامات إتمام الدروس).
- **شريط تقدّم القراءة** + تأثيرات ظهور عند التمرير + تصميم متجاوب بالكامل (RTL).

## خريطة المسارات (URIs)
| المسار | الوصف |
| --- | --- |
| `/` | الصفحة الرئيسية |
| `/course` | صفحة الدورة (كل الوحدات والدروس مع شريط التقدّم) |
| `/module/:id` | صفحة وحدة واحدة (مثال: `/module/m1`) |
| `/lesson/:id` | صفحة درس واحد (مثال: `/lesson/m1l1`) — محتوى + أنشطة + اختبار |
| `/mindmaps` | الخرائط الذهنية والمشجرات |
| `/about` | عن المجموعة ومحاورها الأربعة |
| `/static/*` | الأصول الثابتة (الشعار، CSS، JS) |

معرّفات الدروس: `m1l1..m1l3`, `m2l1..m2l3`, `m3l1..m3l3`, `m4l1..m4l3`.

## بنية البيانات
- **المحتوى التعليمي**: معرّف بالكامل في `src/data/course.ts` (Modules → Lessons → Sections/Exercises/Quiz + MindMaps + Goals).
- **التخزين**: تقدّم المستخدم يُحفظ على المتصفّح عبر `localStorage` (مفتاح `sk_progress_v1`) — لا حاجة لقاعدة بيانات خادمية.
- **تدفّق البيانات**: بيانات ثابتة (static) تُعرض عبر Hono SSR (JSX) → HTML، والتفاعل من جانب العميل عبر `public/static/app.js`.

## دليل الاستخدام
1. افتح الصفحة الرئيسية وتعرّف على الأهداف والمميزات.
2. اضغط **«ابدأ الدورة»** للانتقال إلى صفحة الدورة ورؤية كل الوحدات.
3. ادخل أي درس: اقرأ المحتوى، نفّذ الأنشطة، ثم أجب على الاختبار القصير.
4. اضغط **«أنهيت هذا الدرس»** (أو أكمل القراءة) لتُحتسب ضمن تقدّمك.
5. راجع **الخرائط الذهنية** لتثبيت المفاهيم بصريًّا.

## التقنيات
- **Hono** (إطار خفيف على Cloudflare) + **JSX SSR**.
- **Vite** للبناء، **Wrangler** للتطوير/النشر.
- **Tailwind غير مستخدم** — نظام CSS مخصّص (`public/static/style.css`) + Font Awesome + خطوط Google (Tajawal/Cairo).

## التشغيل محليًا

